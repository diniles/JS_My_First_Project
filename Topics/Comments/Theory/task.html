Comments

There is a way in JS to write text pieces that the JS interpreter will ignore. Such elements are called comments. Comments make code more readable and comprehensible for developers, even though the interpreter will ignore them.

JavaScript language supports two types of comments: single-line and multi-line. Let's look at them more closely.

Single-line comments

Single-line comments refer to comments that only occupy one line. Take a careful look at the syntactic features of these comments:

console.log("Nice to see you!"); // This code outputs the message to the console

In the example above, the text after // is ignored by the interpreter. With the help of such notes, you will recollect what a specific code fragment is intended for, even if it's been a while.

Multi-line comments

Comments that take up several lines are called multi-line comments. These are their syntactic features:

/*  
  The following code outputs the message to the console
  The console will display a line with the text "Hello, JS!"
*/
console.log("Hello, JS!");

The interpreter ignores all text between /* and the nearest */. Such comments help explain complex code pieces or temporarily comment out code fragments when you need to find an error. Also, multi-line comments at the beginning of a file are pretty useful when you specify copyright information.

Sometimes, you may find a slightly modified syntax of multi-line notes: the comment opens with the /** tag, and each line starts with an *:

/**
* The first program
* Author: Bob  
*/
console.log("Hi, I'm Bob");

Such comments often contain information about the programming file, including its name, version, and the script author.

JavaScript does not support nested comments. This means that you cannot write a comment inside another comment.

Conclusion

Comments make your code more transparent and help simplify the work of all those programmers who will work with your code in the future. Comments also temporarily comment out code fragments when you need to find an error. In other words, you can benefit from comments, so don't ignore them, even though your JS interpreter will. Do not overuse comments; use them wisely, and do not forget that comments can't replace well-written code.
